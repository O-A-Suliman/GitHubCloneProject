"أنا أعمل على مشروع يستخدم Django (Rest Framework) في الـ Backend و React (Vite) في الـ Frontend. قمت برفع ملفات المشروع لك. أريدك أن تقوم بتحليل الأكواد الحالية وإجراء التعديلات التالية:

في جانب Django (Backend):

قم بتهيئة مكتبة django-cors-headers في ملف settings.py للسماح لـ React (الذي يعمل على http://localhost:5173) بالوصول للـ API.

قم بتأمين الإعدادات: اجعل DEBUG = False (مع توضيح كيفية تبديلها)، وأضف إعدادات الـ ALLOWED_HOSTS.

في ملفات الـ serializers.py الخاصة بتطبيق الـ blog تطبيق، تأكد من جعل حقل الكاتب (author) كـ read_only لمنع التلاعب به من قبل المستخدمين.

في جانب React (Frontend):

راجع ملف api.js وتأكد من أن الـ baseURL مهيأ بشكل صحيح للاتصال بـ Django.

أضف interceptor لـ Axios يقوم بإلحاق توكن التحقق (JWT/Token) تلقائياً في الـ Header لكل الطلبات الصادرة.

أضف معالجة بسيطة للأخطاء (Error Handling) في حال كان السيرفر متوقفاً.

إدارة المشروع:

أعطني محتوى ملف .gitignore مثالي لهذا المشروع لاستبعاد مجلدات __pycache__ و node_modules وقاعدة بيانات sqlite.

المطلوب: تزويدي بالأكواد المعدلة لكل ملف على حدة مع شرح بسيط للأماكن التي يجب وضع الكود فيها."

لماذا هذا البرومبت قوي؟
محدد التقنيات: يخبر النموذج بالأدوات المستخدمة (Vite, DRF).

يركز على نقاط الضعف: يوجه الذكاء الاصطناعي مباشرة لمشكلة الـ CORS والتوكن.

منظم: يطلب الأكواد بشكل مرتب ملفاً بملف، مما يسهل عليك عملية النسخ واللصق.
